# OBLIGATORIO:

- Siempre debes leer la carpeta docs para entender el proyecto con su documentación.
- La documentación se debe seguir al pie de la letra.
- Usa siempre Context7 MCP para verificar la sintaxtis de Next.js, PrismaORM y los que haya falta antes de generar código.

# REGLAS

- Usa siempre Tailwind y no escribas CSS manual.
- Aplica siempre DRY y buenas prácticas de código.
- Crea interfaces y types en archivos separados.
- Usa una carpeta dedicada para archivos de tipado.
- Mantén componentes pequeños, máximo 120 líneas.
- Usa nombres claros y consistentes en archivos, funciones y variables.
- Evita funciones gigantes; divide lógica en módulos reutilizables.
- Prioriza composición sobre duplicación de componentes.
- Usa tipado estricto en TypeScript; evita `any`.
- Valida entradas y maneja errores de forma explícita.
- Mantén una estructura de carpetas predecible y estable.
- Separa UI, lógica de negocio y acceso a datos.
- Escribe funciones puras cuando sea posible.
- Evita efectos secundarios ocultos.
- Documenta decisiones importantes de arquitectura en texto breve.
- Mantén imports ordenados y elimina código muerto.
- No dejes comentarios obsoletos ni TODOs sin contexto.
- Garantiza accesibilidad básica en la interfaz.
- Verifica que la UI funcione en móvil y escritorio.
- Antes de cerrar cambios, confirma lint limpio y build exitosa.
